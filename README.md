# Knowledge4Graph

Multi-project knowledge graph dashboard. Visualize and explore codebase architectures generated by [understand-anything](https://github.com/nicobailey/understand-anything).

## Quick Start

```bash
pnpm install
pnpm dev
```

Open `http://localhost:5173` to see the project selector.

## Import a Project

1. Run `/understand` on your project to generate a knowledge graph
2. Import it:

```bash
python3 scripts/import-project.py <project-id> /path/to/project
```

This copies the graph data, and for projects under 100MB, bundles source files for remote preview.

Options:
- `--skip-source` — skip source file extraction
- `--size-limit 50` — set size threshold in MB (default: 100)
- `--name "My Project"` — display name
- `--description "..."` — project description

## Build for Deployment

```bash
pnpm build
```

Output in `packages/dashboard/dist/` — deploy as static files to Cloudflare Pages, Vercel, or any static host.

## Project Structure

```
packages/
├── core/dist/       # Pre-built schema validation & search (from understand-anything)
└── dashboard/       # React dashboard (Vite + Tailwind + xyflow)
    ├── src/         # Source code
    └── public/
        └── graphs/  # Knowledge graph data per project
            ├── index.json
            └── {project-id}/
                ├── knowledge-graph.json
                ├── files/           # Bundled source previews (optional)
                ├── meta.json
                └── config.json
scripts/
└── import-project.py  # Import tool
```

## Adding Projects

Each project is a directory under `public/graphs/` with a `knowledge-graph.json`. Register it in `public/graphs/index.json`:

```json
[
  {
    "id": "my-project",
    "name": "My Project",
    "description": "What this project does",
    "hasSource": true,
    "sourceMode": "bundled"
  }
]
```
